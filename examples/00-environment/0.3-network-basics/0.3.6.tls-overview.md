# 0.3.6 TLS / HTTPS 观察清单

对应笔记：[0.3.6 TLS/SSL](../../../notes/00-environment/0.3-network-basics/0.3.6.tls-ssl.md)

## 步骤

1. 访问任意 `https://` 网站。
2. 点击地址栏锁图标 → **连接是安全的** → **证书有效**（文案因浏览器略异）。
3. 查看：颁发给（域名）、颁发者、有效期。
4. DevTools → **Security** 面板：确认 TLS 版本（常见 TLS 1.2 / 1.3）、证书摘要。
5. Network → Protocol 常为 `h2`/`h3`（TLS 之上）。

## 对比实验（可选）

| 操作 | 预期 |
| --- | --- |
| 访问纯 `http://` 站点 | 地址栏提示「不安全」 |
| HTTPS 页强制加载 `http://` 图片/脚本 | 控制台混合内容警告或拦截 |
| Cookie 未加 `Secure` 却指望仅 HTTPS | 设计上应在 HTTPS 使用 `Secure` |

## 记住

- HTTPS 保护 **传输**；XSS、越权仍可能存在。
- 生产 WebSocket 用 **`wss://`**。
- 口语「SSL 证书」通常指 TLS 证书。
