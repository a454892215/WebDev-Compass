# 0.2.4 Git 工作流速查

对应笔记：[0.2.4 Git 基础](../../../notes/00-environment/0.2-dev-environment/0.2.4.git.md)

---

## 首次与 clone

```bash
git config --global user.name "Your Name"
git config --global user.email "you@example.com"
git clone <url>
cd <repo>
```

---

## 日常循环

```bash
git status
git pull origin main
git checkout -b feature/my-change
# … 编辑文件 …
git add .
git commit -m "feat: describe change"
git push -u origin feature/my-change
# → 在 GitHub 开 Pull Request
```

---

## 合并与整理

```bash
# merge（在 main 上）
git checkout main
git pull origin main
git merge feature/my-change

# rebase（在 feature 上，推送前整理）
git checkout feature/my-change
git fetch origin
git rebase origin/main
git push --force-with-lease   # 仅当该分支仅你一人使用时
```

---

## 冲突与撤销

```bash
# 解决冲突后
git add <resolved-files>
git commit

# 撤销工作区
git restore <file>

# 暂存当前改动
git stash
git stash pop
```

---

## PR 检查清单

- [ ] 从最新 `main` 拉分支
- [ ] commit message 清晰（feat/fix/docs…）
- [ ] 不含 `node_modules`、`.env` 等
- [ ] PR 描述：改了什么 + 如何验证
- [ ] CI 通过后再 merge

---

## 常用查看

```bash
git log --oneline --graph -10
git diff
git diff --staged
git branch -a
git remote -v
```
