# 0.2.2 熟练使用 VS Code / Cursor 及 Web 相关插件

> [← 返回 0.2 目录](./README.md) · [知识库地图](../../../knowledge-map/01-foundation-stages-0-2.md#02-开发环境)

## 一句话理解

**VS Code / Cursor** 是 Web 开发的主战场：写代码、跑终端、调 Git、装插件扩展能力；配合 **ESLint、Prettier、Live Server** 等插件，形成「写 → 查 → 格式化 → 预览」的高效闭环。

---

## VS Code 与 Cursor 的关系

| | VS Code | Cursor |
| --- | --- | --- |
| 基础 | 微软开源编辑器 | 基于 VS Code 分支 |
| 插件 | VS Code Marketplace 几乎全部兼容 | 同上 |
| 配置 | `settings.json`、`.vscode/` | 相同 |
| 差异 | 纯编辑器 | 内置 AI 辅助（Chat、Composer、Tab 补全） |

**学习重点**：界面、快捷键、插件、工作区配置 **通用**。本笔记以 Cursor 为例，VS Code 用户操作一致。

## ESLint：代码质量守门员

ESLint 根据规则检查：未使用变量、错误比较、`==` vs `===`、React Hooks 依赖等。

### 典型工作流

1. 项目根目录有 `eslint.config.js`（或旧版 `.eslintrc.cjs`）
2. 安装 ESLint 插件后，编辑器 **实时** 在问题面板与行内显示
3. 保存时可配置 **自动 fix** 可修复项

### 用户级 vs 工作区

| 级别 | 文件 | 优先级 |
| --- | --- | --- |
| 工作区 | `.vscode/settings.json` | **高**（团队共享） |
| 用户 | Cursor Settings | 低（个人偏好） |

工作区推荐片段（见配套示例）：

```json
{
  "editor.codeActionsOnSave": {
    "source.fixAll.eslint": "explicit"
  },
  "eslint.validate": ["javascript", "javascriptreact", "typescript", "typescriptreact"]
}
```

## Prettier：格式化统一

ESLint 管 **对错**；Prettier 管 **长相**（缩进 2 还是 4、单引号还是双引号）。

### 避免与 ESLint 冲突

现代项目用 `eslint-config-prettier` 关闭 ESLint 中与 Prettier 重叠的规则，两者分工：

- **ESLint**：逻辑问题
- **Prettier**：格式

### 保存时格式化

```json
{
  "editor.defaultFormatter": "esbenp.prettier-vscode",
  "editor.formatOnSave": true
}
```

对特定语言单独指定 formatter 时，用 `[javascript]`、`[typescript]` 等语言块。

## Live Server：本地预览

直接用 `file://` 打开 HTML 时：

- 部分 API（如 `fetch` 本地模块、Service Worker）行为受限
- 与生产环境的 `http://` 不一致

**Live Server** 在本地启动 HTTP 服务（默认 `http://127.0.0.1:5500`），保存文件后自动刷新。

### 使用步骤

1. 安装 Live Server 插件
2. 右键 HTML 文件 → **Open with Live Server**
3. 或命令面板：`Live Server: Open with Live Server`

### 替代方案

```bash
npx serve .
npx vite --open
python3 -m http.server 8080
```

阶段 3 后会主要用 **Vite dev server**，Live Server 适合纯静态 HTML 入门。

## 工作区配置 `.vscode/`

团队项目常在仓库根目录提交：

```
.vscode/
├── extensions.json   # 推荐插件（同事打开项目会提示安装）
├── settings.json     # 工作区编辑器设置
└── launch.json       # 调试配置（可选）
```

**extensions.json** 示例：

```json
{
  "recommendations": [
    "dbaeumer.vscode-eslint",
    "esbenp.prettier-vscode",
    "ritwickdey.liveserver",
    "editorconfig.editorconfig"
  ]
}
```

好处：新人 clone 仓库后，Cursor 提示「是否安装推荐扩展」，环境一致。

## EditorConfig

`.editorconfig` 与编辑器无关，定义基础格式：

```ini
root = true

[*]
charset = utf-8
end_of_line = lf
indent_style = space
indent_size = 2
insert_final_newline = true
trim_trailing_whitespace = true
```

Prettier 可读取 EditorConfig；多编辑器协作时很有用。

## 实践

1. 打开配套配置目录，复制 `.vscode/`、`.prettierrc`、`.editorconfig` 到你的练习项目根目录
2. 安装推荐扩展，打开任意 `.html` → Live Server 预览
3. 故意写一行 `var x = 1`（若 ESLint 规则禁止 var），观察 Problems 面板提示
4. `Shift + Option + F` 格式化，观察 Prettier 效果

## 常见误区

| 误区 | 正确理解 |
| --- | --- |
| 「格式化靠手动调缩进」 | 应用 Prettier + formatOnSave，团队零争论 |
| 「ESLint 和 Prettier 重复了」 | 用 eslint-config-prettier 分工，不重复 |
| 「Live Server 就是生产部署」 | 仅本地开发；上线用 Nginx/CDN 等 |
| 「用户 settings 要提交 Git」 | 个人偏好放用户设置；团队规范放 `.vscode/` |
| 「Cursor 只能用它自带的 AI」 | 插件、配置与 VS Code 生态基本通用 |
