# 0.2.3 熟练使用 Chrome DevTools

> [← 返回 0.2 目录](./README.md) · [知识库地图](../../../knowledge-map/01-foundation-stages-0-2.md#02-开发环境)

## 一句话理解

**Chrome DevTools** 是内建在 Chrome 里的开发者工具：让你能 **看见** DOM 与样式、**执行** JavaScript、**追踪** 网络请求、**调试** 源码、**分析** 性能、**检查** 存储——前端排错 80% 的时间都在这里。

---

## 如何打开

| 方式 | 说明 |
| --- | --- |
| `F12` 或 `Cmd + Option + I`（Mac） | 打开/关闭 DevTools |
| 右键页面 → **检查（Inspect）** | 打开并定位到元素 |
| `Cmd + Shift + C` | 元素选择器（点击页面元素跳转到 Elements） |

建议：**Dock 位置** 设为底部或独立窗口，按习惯调整（Settings → Preferences → Appearance）。

---

## 面板总览

```
Elements | Console | Sources | Network | Performance | Memory | Application | ...
```

本节重点六个：**Elements、Console、Network、Sources、Performance、Application**。

---

## 1. Elements（元素）

### 做什么

- 查看与编辑 **DOM 树**
- 查看与修改 **CSS 样式**（Computed、Styles）
- 查看盒模型、布局、事件监听器

### 常用操作

| 操作 | 说明 |
| --- | --- |
| 选择元素 | 左上角箭头图标，或 `Cmd+Shift+C` |
| 编辑 HTML | 双击节点或右键 → Edit as HTML |
| 临时改样式 | 右侧 Styles 面板增删 CSS（刷新后丢失） |
| 强制状态 | `:hov` → 勾选 `:hover`、`:focus`、`:active` |
| Computed | 查看最终计算后的样式与盒模型尺寸 |

### 调试布局

- **Flexbox / Grid** 可视化：Elements 选中容器后，Styles 旁有 layout 图标
- **盒模型**：margin / border / padding / content 一目了然

### 与代码的关系

Elements 里是 **浏览器渲染后的 DOM**，可能与源码 HTML 不同（JS 动态插入、框架虚拟 DOM 渲染结果）。

---

## 2. Console（控制台）

### 做什么

- 查看 `console.log` / 错误 / 警告
- **交互式执行** JavaScript（当前页面的上下文）
- 查看未捕获异常与网络失败

### 常用 API

```javascript
console.log('普通日志', { id: 1 });
console.warn('警告');
console.error('错误');
console.table([{ name: 'A' }, { name: 'B' }]);
console.time('fetch');
// ... 异步操作
console.timeEnd('fetch');
```

### 技巧

| 技巧 | 说明 |
| --- | --- |
| `$0` | 指 Elements 里当前选中的 DOM 节点 |
| `copy(obj)` | 把对象复制到剪贴板 |
| **Filter** | 按级别过滤 log / warn / error |
| **Preserve log** | 页面跳转后保留日志（Network 也有） |

### 与移动端对照

类似 Android **Logcat**、Xcode **Console**：运行时输出与错误的第一现场。

---

## 3. Network（网络）

### 做什么

记录页面加载与 JS 触发的 **所有 HTTP(S) 请求**：URL、方法、状态码、耗时、请求/响应头与 body。

### 必会选项

| 选项 | 说明 |
| --- | --- |
| **Preserve log** | 跳转 SPA 路由时保留记录 |
| **Disable cache** | 开发时禁用缓存，模拟硬刷新 |
| **Throttling** | 模拟慢 3G，测弱网 |
| **Filter** | XHR/Fetch、JS、CSS、Doc 等类型过滤 |

### 分析单次请求

点击某条请求 → 查看：

- **Headers**：General（URL、Method、Status）、Request Headers、Response Headers
- **Preview / Response**：JSON 或 HTML  body
- **Timing**：DNS、连接、TTFB、下载各阶段耗时

### 与 0.1.1 的联系

「用户点击 → fetch API → 服务器响应」全程在 Network 可见。对接后端时 **Status 200 但业务 code 失败** 也要看 Response body。

---

## 4. Sources（源代码）

### 做什么

- 浏览页面加载的 **JS / HTML / CSS**（含 webpack/vite 打包后的文件）
- **断点调试**：暂停执行、查看变量、单步跟踪
- 编辑本地覆盖（Overrides，高级用法）

### 断点类型

| 类型 | 用法 |
| --- | --- |
| **行断点** | 点击行号 |
| **条件断点** | 右键行号 → Add conditional breakpoint |
| **`debugger` 语句** | 代码里写 `debugger;`，运行到此处自动暂停 |
| **XHR/fetch breakpoint** | 某 URL 请求发出时暂停 |

### 调试控制

- **Resume**（F8）：继续执行
- **Step over**（F10）：单步跳过函数
- **Step into**（F11）：进入函数内部
- **Scope**：查看当前作用域变量
- **Call Stack**：调用栈

### Source Map

生产环境 JS 常被压缩混淆；若项目配置了 **source map**，Sources 里可映射回 **原始 TypeScript/Vue 文件**。

## 5. Performance（性能）

### 做什么

录制一段时间内的 **CPU、渲染、脚本执行、布局、绘制**，找出卡顿原因。

### 基本流程

1. 打开 Performance → 点击 **Record**
2. 在页面上做操作（滚动、点击、动画）
3. Stop → 分析 **Main** 线程火焰图
4. 关注 Long Task（超过 50ms 的任务）、Layout / Paint 频繁

### 何时用

- 页面滚动掉帧
- 首屏加载慢（可配合 Network 的 waterfall）
- 复杂动画或大量 DOM 更新

入门阶段知道 **能录、能看 Main 线程** 即可；阶段 11 性能优化会深入。

## 6. Application（应用）

### 做什么

管理浏览器侧 **持久化与后台能力** 存储：

| 存储 | 说明 |
| --- | --- |
| **Local Storage** | 同源永久 KV，除非手动清除 |
| **Session Storage** | 标签页级，关闭标签即清 |
| **Cookies** | 带过期、HttpOnly、SameSite 等属性 |
| **IndexedDB** | 结构化大容量数据库 |
| **Cache Storage** | Service Worker 缓存 |
| **Manifest / Service Workers** | PWA 相关 |

### 常用场景

- 登录后检查 **Cookie / Token** 是否写入
- 调试 **localStorage** 里的主题、语言设置
- 清除站点数据：**Clear site data** 模拟首次访问

## 推荐调试工作流

```
1. 复现问题
2. Console 看有没有红色报错
3. Network 看请求是否发出、状态码、响应体
4. Elements 看 DOM / CSS 是否符合预期
5. Sources 下断点跟逻辑
6. Performance 仅在卡顿时使用
7. Application 查存储与 Cookie
```

## 📱 移动端对照

| Chrome DevTools | Android | iOS |
| --- | --- | --- |
| Elements | Layout Inspector | View Hierarchy（需工具） |
| Console / Log | Logcat | Xcode Console |
| Network | OkHttp logging / Charles 抓包 | Charles / Proxyman |
| Sources 断点 | Android Studio Debugger | Xcode Debugger |
| Performance | Android Profiler | Instruments |
| Application 存储 | App 数据目录 / SharedPreferences | UserDefaults / Keychain |

Web 优势：**浏览器自带**，无需额外装 Charles 即可看 Network（HTTPS 本地页面仍可见，生产抓包第三方 App 有时需证书）。

**远程调试**：Chrome 可 `chrome://inspect` 调试 Android WebView；Safari 开发菜单调试 iOS WKWebView。

## 常见误区

| 误区 | 正确理解 |
| --- | --- |
| 「改 Elements 里样式就是改代码了」 | 只是临时预览，需改源文件 |
| 「Console 里能跑就说明生产没问题」 | Console 环境、权限、数据可能与用户不同 |
| 「Network 只有 Failed 才要看」 | 200 也可能 body 里 business error |
| 「禁用缓存只在 Network 勾一次就行」 | 开发期建议常开 Disable cache |
| 「Performance 每次都要录」 | 仅在感知卡顿时分析，避免过度优化 |
