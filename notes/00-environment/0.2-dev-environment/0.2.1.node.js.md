# 0.2.1 安装与配置：Node.js、npm / yarn / pnpm

> [← 返回 0.2 目录](./README.md) · [知识库地图](../../../knowledge-map/01-foundation-stages-0-2.md#02-开发环境)

## 一句话理解

**Node.js** 让 JavaScript 能在电脑上脱离浏览器运行；**npm / yarn / pnpm** 是管理第三方库与脚本命令的包管理器——现代 Web 前端工程化几乎全都建立在它们之上。

---

## 为什么 Web 前端需要 Node.js

浏览器里的 JavaScript **不能**直接做这些事：

- 读写项目文件夹、启动本地开发服务器
- 运行 Vite / Webpack 打包 TypeScript 与 Vue/React 单文件组件
- 执行 ESLint、Prettier、单元测试

因此需要 **Node.js**：基于 Chrome V8 引擎的 JavaScript **运行时**，把 JS 当作服务端/工具链语言来用。

```
你的源代码 (.ts / .vue / .jsx)
        │
        ▼
  Node.js 上的构建工具 (Vite / Webpack)
        │
        ▼
  浏览器可运行的产物 (HTML / JS / CSS)
```

> 注意：Node.js ≠ 浏览器。DOM API（`document`、`window`）在 Node 里默认不存在。

## 安装 Node.js

### 推荐方式

| 方式 | 适用场景 | 说明 |
| --- | --- | --- |
| **nvm**（Node Version Manager） | macOS / Linux 开发 | 可切换多版本，团队对齐 LTS 版本 |
| **fnm** | 跨平台，速度快 | Rust 实现，用法类似 nvm |
| **官网安装包** | Windows 或快速入门 | [nodejs.org](https://nodejs.org/) 下载 LTS |

### 版本选择

- 选 **LTS（Long Term Support）**，不要追最新的 Current 除非有明确需求
- 团队项目看 `package.json` 的 `engines` 字段，例如 `"node": ">=20"`

### 安装后验证

```bash
node -v    # 例如 v20.x.x
npm -v     # 例如 10.x.x（npm 随 Node 自带）
```

## npm：默认包管理器

**npm（Node Package Manager）** 随 Node 一起安装，负责：

- 从 [npm registry](https://www.npmjs.com/) 下载开源包
- 管理项目依赖（写入 `package.json`）
- 运行脚本（`npm run dev` / `npm run build`）

### 核心文件

| 文件 | 作用 |
| --- | --- |
| `package.json` | 项目元信息、依赖声明、scripts |
| `package-lock.json` | 锁定依赖的精确版本（npm 生成） |
| `node_modules/` | 实际安装的包（通常不提交 Git） |

### 常用命令

```bash
npm init -y                 # 初始化 package.json
npm install                 # 按 lock 文件安装依赖
npm install axios           # 安装生产依赖
npm install -D vite         # 安装开发依赖（devDependencies）
npm run dev                 # 运行 scripts 里定义的 dev
npm uninstall lodash        # 移除依赖
npx create-vite@latest .    # 临时下载并执行脚手架（无需全局安装）
```

### `dependencies` vs `devDependencies`

| 类型 | 何时需要 | 示例 |
| --- | --- | --- |
| **dependencies** | 产物运行时需要 | `react`、`axios` |
| **devDependencies** | 仅开发/构建时需要 | `vite`、`typescript`、`eslint` |

## yarn：Facebook 推出的替代方案

特点：早期以并行安装更快著称；现常用 **Yarn Berry（v2+）** 或经典 **Yarn Classic（v1）**。

```bash
# 启用 Corepack（Node 16.10+ 自带，推荐）
corepack enable
corepack prepare yarn@stable --activate

yarn --version
yarn install
yarn add axios
yarn add -D vite
yarn dev
```

锁文件：`yarn.lock`

---

## pnpm：省磁盘、严格依赖

**pnpm** 用全局内容寻址存储 + 硬链接，多个项目共享同一份包，**省空间**；且默认 **不允许** 依赖访问未声明的包（更严格）。

```bash
corepack enable
corepack prepare pnpm@latest --activate

pnpm -v
pnpm install
pnpm add axios
pnpm add -D vite
pnpm dev
```

锁文件：`pnpm-lock.yaml`

Monorepo 场景下 pnpm workspace 很流行（阶段 13 会深入）。

## 三者怎么选

| | npm | yarn | pnpm |
| --- | --- | --- | --- |
| 上手成本 | 最低（Node 自带） | 低 | 低 |
| 磁盘占用 | 较高 | 较高 | **较低** |
| 严格性 | 一般 | 一般 | **更严格** |
| 生态兼容 | 最好 | 好 | 好（绝大多数项目可用） |

**建议**：跟团队统一即可。个人学习可用 **npm** 或 **pnpm**；阅读开源项目时看锁文件类型（`package-lock.json` / `yarn.lock` / `pnpm-lock.yaml`）。

## `package.json` 关键字段

```json
{
  "name": "my-web-app",
  "version": "1.0.0",
  "private": true,
  "type": "module",
  "scripts": {
    "dev": "vite",
    "build": "vite build",
    "preview": "vite preview"
  },
  "dependencies": {
    "axios": "^1.7.0"
  },
  "devDependencies": {
    "vite": "^5.4.0"
  },
  "engines": {
    "node": ">=20"
  }
}
```

| 字段 | 说明 |
| --- | --- |
| `"type": "module"` | 使用 ES Modules（`import`/`export`） |
| `scripts` | 自定义命令别名，本质是 shell 命令 |
| `^1.7.0` | SemVer：允许兼容的次版本升级（阶段 3.1 详述） |

## 📱 移动端对照

| Web | Android | Flutter | React Native |
| --- | --- | --- | --- |
| Node.js | JDK + Android SDK | Dart SDK | Node.js（同样用于 Metro） |
| npm / pnpm | Gradle 依赖、Maven | pub.dev / `pubspec.yaml` | npm / yarn |
| `package.json` | `build.gradle` / `libs.versions.toml` | `pubspec.yaml` | `package.json` |
| `node_modules/` | Gradle 缓存目录 | `.pub-cache` | `node_modules/` |
| `npm run dev` | Run App（Gradle 构建 + 安装） | `flutter run` | `npx react-native start` |

**核心一致**：都是「声明依赖 → 工具下载 → 本地构建 → 运行」。Web 的差异在于产物给 **浏览器** 用，而不是 APK/IPA。

## 常见误区

| 误区 | 正确理解 |
| --- | --- |
| 「前端只需要浏览器，不用装 Node」 | 现代前端 **工程化** 几乎都需要 Node 跑构建与 dev server |
| 「Node 版本无所谓」 | 不同 major 版本 API 有差异，团队应统一 LTS |
| 「把 node_modules 提交到 Git」 | 应忽略；用 lock 文件 + `npm ci` 复现 |
| 「npm 和 npx 是一回事」 | `npm` 管理包；`npx` 临时执行包内命令 |
| 「yarn 和 pnpm 可以完全混用同一项目的 lock」 | 应选定一种包管理器，不要混用多种 lock 文件 |
