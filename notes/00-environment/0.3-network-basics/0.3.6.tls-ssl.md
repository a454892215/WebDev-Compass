# 0.3.6 TLS/SSL 与 HTTPS 加密的基本原理

> [← 返回 0.3 目录](./README.md) · [知识库地图](../../../knowledge-map/01-foundation-stages-0-2.md#03-网络基础web-开发者必备)

## 一句话理解

**HTTPS = HTTP + TLS**。TLS（前身常被称作 SSL）在传输层之上提供 **加密、完整性校验、服务器身份认证**，防止窃听、篡改与仿冒站点。

---

## SSL 与 TLS

| 名称 | 说明 |
| --- | --- |
| **SSL** | 旧称；SSL 3.0 已废弃 |
| **TLS** | 现行标准；常见 TLS 1.2 / 1.3 |
| 口语「SSL 证书」 | 实际多为 **TLS 证书** |

前端业务代码不实现 TLS；由浏览器与服务器（或 CDN）完成握手与加解密。

---

## HTTPS 解决的三个问题

| 能力 | 没有 HTTPS 时 | 有 TLS 时 |
| --- | --- | --- |
| **保密性** | 中间人可读 Cookie、表单、JSON | 传输内容加密 |
| **完整性** | 内容可被篡改（插广告、改脚本） | 篡改会被发现 |
| **身份认证** | 你可能连到假冒服务器 | 证书证明域名归属 |

> 注意：HTTPS **不**自动等于网站安全——XSS、弱口令、错误鉴权仍可能存在。它保护的是 **传输通道**。

---

## 证书与信任链（概念）

```
浏览器信任的根 CA
        ↓ 签发
 intermediate CA
        ↓ 签发
  example.com 的站点证书（含公钥、域名、有效期）
```

握手时服务器出示证书；浏览器校验：

1. 证书链是否锚到信任的根 CA
2. 域名是否匹配（SAN）
3. 是否在有效期内、未被吊销（简化理解）

失败时出现「您的连接不是私密连接」等警告——开发环境自签证书也是同类机制。

---

## 握手在做什么（极简）

以 TLS 1.3 直观理解：

```
1. 客户端与服务器协商密码套件、交换密钥材料
2. 验证服务器证书
3. 双方得出共享会话密钥（对称加密用）
4. 之后 HTTP 报文在加密通道上传输
```

- **非对称加密**（证书公钥）：主要用于握手阶段的身份与密钥交换
- **对称加密**（会话密钥）：用于后续大数据量传输（更快）

TLS 1.3 相比 1.2：往返更少、废弃一批不安全算法，现代浏览器优先使用。

## 与前端相关的实践点

| 点 | 说明 |
| --- | --- |
| **混合内容** | HTTPS 页面请求 HTTP 图片/脚本会被拦截或警告 |
| **Secure Cookie** | 仅在 HTTPS 下发送 |
| **HSTS** | 响应头强制浏览器后续只用 HTTPS |
| **本地开发** | `localhost` 常被视为安全上下文；真机调设备需 HTTPS 或特殊配置 |
| **wss://** | WebSocket 的 TLS 版；HTTPS 页应使用 `wss` |

安全上下文（Secure Context）下才能用：部分 Geolocation、Service Worker、强加密相关 API 等。

---

## 证书常见类型（了解）

| 类型 | 说明 |
| --- | --- |
| **DV** | 域名验证，最常见、自动化（Let's Encrypt） |
| **OV / EV** | 组织验证；现代浏览器 UI 差异已弱化 |
| **通配符** | `*.example.com` |
| **自签** | 开发用；浏览器不默认信任 |

部署上多用 CDN / 网关 / Certbot 自动续期，前端较少手管私钥。

## 📱 移动端对照

| Web | 移动端 |
| --- | --- |
| 浏览器校验证书链 | OkHttp / URLSession 默认校验 HTTPS |
| 用户看到证书警告页 | App 证书错误通常直接失败 |
| 证书钉扎（Pinning）少见 | 金融类 App 更常做 SSL Pinning |
| 混合内容策略 | App 内 WebView 同样需注意混合内容 |

## 常见误区

| 误区 | 正确理解 |
| --- | --- |
| 「HTTPS = 网站没有漏洞」 | 只保证传输；应用层安全另说 |
| 「HTTP 内网就够了」 | 内网也有嗅探与误配风险；敏感系统仍建议 TLS |
| 「前端要自己加密 JSON」 | 传输靠 TLS；业务层加密是额外需求，不是替代 HTTPS |
