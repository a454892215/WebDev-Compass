# 0.3.5 DNS 解析与 CDN 基本概念

> [← 返回 0.3 目录](./README.md) · [知识库地图](../../../knowledge-map/01-foundation-stages-0-2.md#03-网络基础web-开发者必备)

## 一句话理解

**DNS** 把域名翻译成 IP，浏览器才能连上服务器；**CDN** 把静态资源缓存在离用户更近的节点，降低延迟并减轻源站压力。

---

## DNS 解析

### 为什么需要 DNS

人记 `www.example.com`，机器连 `93.184.216.34`。DNS（Domain Name System）是这本「全球电话簿」。

### 解析链路（简化）

```
浏览器/OS 缓存 → 路由器/本机配置的 DNS（如 8.8.8.8）
       → 递归解析器查询：
         根 → .com 顶级域 → example.com 权威 DNS
       → 得到 A/AAAA 记录（IPv4/IPv6）→ 返回 IP
```

典型记录类型：

| 类型 | 含义 |
| --- | --- |
| **A** | 域名 → IPv4 |
| **AAAA** | 域名 → IPv6 |
| **CNAME** | 别名 → 另一域名（CDN 常用） |
| **MX** | 邮件服务器 |
| **TXT** | 文本（验证域名、SPF 等） |
| **NS** | 该域的权威 DNS 服务器 |

### 与 Web 前端的关系

1. 首屏时间含 **DNS Lookup**（DevTools Timing 可见）
2. `preconnect` / `dns-prefetch` 可提前解析关键第三方域名
3. 换 DNS / 主机 Hosts 可排查「只有某网络打不开」类问题
4. TTL 决定缓存多久；改解析需等 TTL 生效（切流、发布常遇）

命令行（了解）：

```bash
dig example.com +short
nslookup example.com
```

## CDN（Content Delivery Network）

### 做什么

```
用户 ──近──► CDN 边缘节点（缓存的 JS/CSS/图片/视频）
                │ 未命中或动态请求
                ▼
              源站 Origin
```

- **就近**：地理或网络拓扑上靠近用户
- **缓存**：按 URL + Cache-Control / CDN 规则
- **卸载**：大流量静态资源不打爆源站
- **可选**：HTTPS 终结、WAF、防盗链、压缩

### 前端如何用 CDN

| 用法 | 示例 |
| --- | --- |
| 静态资源域名 | `https://cdn.example.com/app.abc123.js` |
| 公共库 | `https://cdn.jsdelivr.net/npm/vue@3/...`（生产需评估可控性） |
| 图片/视频 | 对象存储 + CDN（OSS/S3 + CloudFront 等） |

带 **内容哈希** 的文件名（`app.a1b2c3.js`）可长缓存：`Cache-Control: max-age=31536000, immutable`。

### CDN 与 DNS

CDN 常给业务域名配 **CNAME** 到 CDN 供应商域名；解析时经 **智能 DNS / Anycast** 落到最近 PoP。对前端而言：仍是访问同一个资源 URL，背后 IP 可能因地区而变。

## 一次页面加载里的位置

```
输入 URL
  → DNS 解析域名得 IP
  → TCP/QUIC + TLS 握手（HTTPS）
  → HTTP 请求 HTML
  → 解析后请求 CDN 上的 CSS/JS/图片（可能又是不同域名 → 再 DNS）
  → API 请求源站或 API 网关
```

优化思路：减少独特域名数量（过多域名增加 DNS + 连接成本；HTTP/2 后少用盲目域名分片）、对关键源 `preconnect`。

## 开发与排障要点

| 现象 | 可能原因 |
| --- | --- |
| 某地区慢、其他地区快 | CDN 节点或跨网质量差 |
| 改了文件用户仍见旧版 | CDN / 浏览器缓存未刷新；未改哈希文件名 |
| 只有公司网能开 / 不能开 | 内网 DNS、Hosts、防火墙 |
| `net::ERR_NAME_NOT_RESOLVED` | DNS 失败：域名错、未备案解析、本地 DNS 异常 |

| Web | 移动端 |

## 常见误区

| 误区 | 正确理解 |
| --- | --- |
| 「上了 CDN 一定更快」 | 配置错误、回源多、节点差时可能更慢 |
| 「CDN 只加速静态」 | 也可代理动态，但收益与配置不同；静态最典型 |
| 「改 DNS 立刻全球生效」 | 受 TTL 与各级缓存影响 |
