# 0.3.1 URL 结构：协议、域名、端口、路径、查询参数、锚点

> [← 返回 0.3 目录](./README.md) · [知识库地图](../../../knowledge-map/01-foundation-stages-0-2.md#03-网络基础web-开发者必备)

## 一句话理解

**URL（Uniform Resource Locator）** 是定位网络资源的标准地址；浏览器的每一次请求、前端路由、API 调用，本质上都是在构造、解析与跳转 URL。

---

## 完整结构

```
https://user:pass@www.example.com:443/path/to/page?q=vue&page=1#section-2
└─┬──┘ └───┬───┘ └──────┬──────┘└┬┘└────┬────┘└─────┬─────┘└────┬────┘
 协议   用户信息（少用）  主机(域名) 端口   路径      查询参数      锚点
```

| 组成部分 | 英文 | 作用 | 示例 |
| --- | --- | --- | --- |
| **协议** | scheme | 用什么方式访问 | `https`、`http`、`ws`、`wss`、`file` |
| **主机** | host | 服务器域名或 IP | `www.example.com`、`127.0.0.1` |
| **端口** | port | 主机上的服务入口 | `443`（HTTPS 默认）、`80`（HTTP）、`5173`（Vite） |
| **路径** | pathname | 资源在服务器上的位置 | `/api/users`、`/docs/index.html` |
| **查询参数** | search / query | 键值对参数 | `?q=vue&page=1` |
| **锚点** | hash / fragment | 页内定位，**不发往服务器** | `#section-2` |

> **Origin（源）** = 协议 + 主机 + 端口。同源策略只看这三项，见 [0.1.4](../0.1-what-is-web-app/0.1.4.same-origin-policy.md)。

## 各部分详解

### 协议（Scheme）

| 协议 | 用途 |
| --- | --- |
| `http://` | 明文 HTTP |
| `https://` | HTTP over TLS，生产默认 |
| `ws://` / `wss://` | WebSocket / 安全 WebSocket |
| `file://` | 本地文件（许多 API 受限） |

默认端口可省略：`https://example.com` ≡ `https://example.com:443`。

### 域名与端口

- **域名**：人类可读名称，经 DNS 解析为 IP（见 [0.3.5](./0.3.5.dns.md)）。
- **IP**：也可直接写 `http://192.168.1.10:8080`（开发/内网常见）。
- **端口**：本地开发常显式写 `localhost:5173`；生产 HTTPS 多用 443。

### 路径（Path）

- 以 `/` 开头的绝对路径：`/api/users/42`
- 相对路径：`../img/logo.png`（相对当前 URL）
- 末尾 `/` 有时表示「目录」，规范上尽量前后端约定一致

### 查询参数（Query）

格式：`?key1=value1&key2=value2`

- 值需 **URL 编码**（空格 → `%20` 或 `+`，中文 → `%E4%...`）
- 适合过滤、分页、搜索；**勿把密码放查询串**（易进日志、Referer）

### 锚点（Hash）

- 仅客户端使用：滚动到元素、`hash` 路由（老式 SPA）
- 修改 hash **不会**触发新的 HTTP 请求
- 服务器收不到 `#` 及之后的内容

## 用浏览器 API 解析 URL

```javascript
const u = new URL('https://example.com:443/api/users?page=2#top');

u.protocol; // "https:"
u.hostname; // "example.com"
u.port;     // ""（默认端口时为空字符串）
u.pathname; // "/api/users"
u.search;   // "?page=2"
u.hash;     // "#top"
u.origin;   // "https://example.com"

u.searchParams.get('page'); // "2"
u.searchParams.set('page', '3');
u.toString(); // 含 page=3 的新 URL
```

拼接相对地址：

```javascript
new URL('/api/users', 'https://example.com').href;
// "https://example.com/api/users"
```

## 绝对 URL vs 相对 URL

| 类型 | 示例 | 何时用 |
| --- | --- | --- |
| 绝对 | `https://cdn.example.com/a.js` | 跨站 CDN、外链 |
| 根相对 | `/api/users` | 同源 API，自动带当前协议+主机 |
| 相对 | `./detail.html` | 同目录静态页 |

前端请求同源 API 优先用 **根相对路径**（`/api/...`），换环境时少改主机名。

## 常见误区

| 误区 | 正确理解 |
| --- | --- |
| 「域名就是 URL」 | 域名只是主机部分；完整 URL 还有协议、路径等 |
| 「hash 会发给服务器」 | 不会；服务器看不到 `#` 之后 |
| 「端口写不写无所谓」 | 非默认端口必须写，否则连错服务 |
