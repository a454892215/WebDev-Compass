# 0.3.2 HTTP/HTTPS 协议：方法、状态码、请求头/响应头

> [← 返回 0.3 目录](./README.md) · [知识库地图](../../../knowledge-map/01-foundation-stages-0-2.md#03-网络基础web-开发者必备)

## 一句话理解

**HTTP** 是浏览器与服务器交换「请求—响应」的应用层协议；**HTTPS** 是跑在 TLS 之上的 HTTP（见 [0.3.6](./0.3.6.tls-ssl.md)）。读懂方法、状态码与消息头，是调试 Network 面板的基本功。

---

## 报文结构

### 请求

```http
GET /api/users?page=1 HTTP/1.1
Host: api.example.com
Accept: application/json
Authorization: Bearer eyJhbG...

（GET 通常无请求体）
```

### 响应

```http
HTTP/1.1 200 OK
Content-Type: application/json; charset=utf-8
Cache-Control: no-store

{"users":[{"id":1,"name":"Ada"}]}
```

| 部分 | 说明 |
| --- | --- |
| **起始行** | 请求：方法 + 路径 + 版本；响应：版本 + 状态码 + 原因短语 |
| **头部** | 元数据，键值对；一行一个 |
| **空行** | 头部与正文的分隔 |
| **正文 Body** | 可选；JSON、HTML、表单、文件等 |

## HTTP 方法

| 方法 | 语义 | 是否幂等 | 是否有 Body | 典型场景 |
| --- | --- | --- | --- | --- |
| **GET** | 获取资源 | 是 | 否（惯例） | 列表、详情、下载 |
| **POST** | 创建/提交 | 否 | 是 | 注册、下单、上传 |
| **PUT** | 整资源替换 | 是 | 是 | 全量更新用户资料 |
| **PATCH** | 部分更新 | 通常否 | 是 | 只改昵称 |
| **DELETE** | 删除资源 | 是 | 可选 | 删帖、注销 |
| **HEAD** | 只要响应头 | 是 | 否 | 探活、检查是否存在 |
| **OPTIONS** | 询问允许的方法 | 是 | 否 | CORS 预检 |

**幂等**：同样请求执行多次，资源最终状态一致。`GET`/`PUT`/`DELETE` 设计上应为幂等；`POST` 每次可能新建一条记录。

> 浏览器地址栏访问、`<a href>` 导航默认是 **GET**。表单 `method="post"` 发 POST。

## 状态码（按百位分类）

| 类别 | 含义 | 常见码 |
| --- | --- | --- |
| **1xx** | 信息 | `100 Continue`（了解即可） |
| **2xx** | 成功 | `200 OK`、`201 Created`、`204 No Content` |
| **3xx** | 重定向 | `301` 永久、`302`/`307` 临时、`304` 未修改（缓存） |
| **4xx** | 客户端错 | `400` 参数、`401` 未认证、`403` 无权限、`404` 不存在、`409` 冲突、`422` 校验失败、`429` 限流 |
| **5xx** | 服务端错 | `500` 内部错误、`502` 网关、`503` 不可用、`504` 网关超时 |

前端处理口诀：

- `2xx` → 按业务成功处理
- `401` → 跳转登录 / 刷新 token
- `403` → 提示无权限
- `404` → 资源不存在页
- `5xx` → 服务异常，可重试或报错

注意：`fetch` **不把 4xx/5xx 当异常**；需 `if (!res.ok) throw ...`。

## 常用请求头 / 响应头

### 请求头

| Header | 作用 |
| --- | --- |
| `Host` | 目标主机（HTTP/1.1 必填） |
| `Content-Type` | 正文类型：`application/json`、`application/x-www-form-urlencoded`、`multipart/form-data` |
| `Accept` | 客户端期望的响应类型 |
| `Authorization` | 鉴权：`Bearer <token>`、`Basic ...` |
| `Cookie` | 自动携带的 Cookie |
| `Origin` / `Referer` | 来源页信息（CORS、安全策略会用到） |
| `Cache-Control` | 缓存指令（如 `no-cache`） |
| `If-None-Match` | 协商缓存（ETag） |

### 响应头

| Header | 作用 |
| --- | --- |
| `Content-Type` | 响应体类型 |
| `Set-Cookie` | 让浏览器存 Cookie |
| `Cache-Control` / `ETag` / `Last-Modified` | 缓存策略 |
| `Location` | 重定向目标（3xx） |
| `Access-Control-*` | CORS 相关 |
| `Strict-Transport-Security` | 强制后续走 HTTPS（HSTS） |

## HTTPS 相对 HTTP

| | HTTP | HTTPS |
| --- | --- | --- |
| 传输 | 明文 | TLS 加密 |
| 默认端口 | 80 | 443 |
| 身份 | 无法验证服务器 | 证书验证域名 |
| 生产 | 仅限内网调试 | **默认必用** |

浏览器在「不安全」的 HTTP 页上会限制：Cookie `Secure`、部分 API、混合内容（HTTPS 页加载 HTTP 资源）等。原理见 [0.3.6](./0.3.6.tls-ssl.md)。

## 前端调用示例

```javascript
// GET
const list = await fetch('/api/users?page=1').then((r) => {
  if (!r.ok) throw new Error(`HTTP ${r.status}`);
  return r.json();
});

// POST JSON
await fetch('/api/users', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    Authorization: `Bearer ${token}`,
  },
  body: JSON.stringify({ name: 'Ada' }),
});
```

用 DevTools **Network** 核对：Method、Status、Request Headers、Response Headers、Payload、Preview。

## 📱 移动端对照

| Web | Android | Flutter | RN |
| --- | --- | --- | --- |
| `fetch` / axios | OkHttp / Retrofit | dio / http | fetch / axios |
| Method + path | `@GET` / `@POST` | `dio.get/post` | 同 Web |

## 常见误区

| 误区 | 正确理解 |
| --- | --- |
| 「POST 更安全」 | 安全性靠 HTTPS + 鉴权；方法只表达语义 |
| 「404 是服务器挂了」 | 4xx 多是客户端问题；5xx 才是服务端 |
| 「fetch 失败 = 网络断了」 | CORS、DNS、TLS 也会失败；4xx/5xx 仍可能 `ok === false` |
